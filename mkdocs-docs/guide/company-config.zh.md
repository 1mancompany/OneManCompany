# 公司配置

三层配置让你塑造 AI 公司的身份和行为 — 同一个操作系统可以运行完全不同的公司。

## 公司方向

你的愿景声明，注入到每位员工的推理上下文中。改变方向，整个公司随之转向。

**控制内容：**

- 构建什么产品
- 瞄准哪个市场
- 战略优先级
- 质量与速度的权衡

**示例：**

> "我们是一家移动优先的游戏工作室，目标用户是 18-35 岁的休闲玩家。我们优先追求精致的用户体验和快速迭代，而非功能完整性。"

每位员工 — 从拆解任务的 COO 到编写代码的工程师 — 都会将这个方向纳入决策考量。

## 公司文化

行为准则，规定每位员工如何工作。同样的 Talent，同样的 Vessel，完全不同的公司个性。

**控制内容：**

- 沟通风格（正式 vs 随意）
- 决策方式（快速行动 vs 深思熟虑）
- 协作规范
- 质量标准

**示例：**

> - 快速发布，更快迭代。MVP 优先于完美。
> - 写初级开发者也能读懂的代码。
> - 每个 PR 必须包含测试，没有例外。
> - 默认异步沟通，只有阻塞时才开会。

## 工作流

`company_rules/` 中的工作流定义控制公司在流程层面如何运作。

**定义内容：**

- 任务如何在组织中流转
- 审批链和质量关卡
- 标准操作流程
- 自动化触发器

工作流以 Markdown 文件定义，由工作流引擎解析，并分发到相应的处理器。

## 配置文件

| 文件 | 控制内容 |
| --- | --- |
| 公司方向文档 | 战略愿景和优先级 |
| 公司文化文档 | 行为规范和标准 |
| `company_rules/*.md` | 工作流定义和标准操作流程 |
| `employees/{id}/profile.yaml` | 个人员工配置 |
| `employees/{id}/guidance.yaml` | 员工专属指导说明 |

## 操作系统类比

这正是 OneManCompany 之所以是操作系统的原因：

- **同一个 iOS** 可以是工作手机，也可以是孩子的游戏机 — 取决于应用和设置
- **同一个 OneManCompany** 可以是游戏工作室，也可以是开发外包公司 — 取决于方向、文化和 Talent

换一套配置，就换了一家公司。
