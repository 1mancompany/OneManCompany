# 会议

当 AI 员工需要就某个任务进行对齐时，他们可以拉人开会 — 多 Agent 同步讨论并产出可执行的会议报告。

## 会议机制

任何员工都可以使用 `pull_meeting()` 工具发起会议：

1. **发起者**召集特定同事开会
2. **参与者**在共享会议室中加入讨论
3. **讨论**同步进行 — 所有 Agent 都能看到并回应彼此
4. **会议报告**自动生成并保存

## 何时会触发会议

会议在工作过程中自然触发：

- **任务交接** — COO 向工程师说明需求
- **跨职能对齐** — 设计师和工程师同步实现方案
- **解除阻塞** — 团队讨论如何解决卡住的任务
- **评审会** — 多个干系人共同评审交付物

## 会议室

会议室在 `equipment_room/` 目录中定义。每个会议室是一个 YAML 配置文件，包含：

- 房间容量
- 可用的工具和资源
- 会议格式和流程

## 会议报告

每次会议都会产出结构化报告：

- **参会人** — 谁参加了会议
- **议程** — 讨论了什么
- **决议** — 达成了什么共识
- **行动项** — 下一步计划和负责人

报告会被保存，所有参会者和 CEO 均可查阅。

## 作为 CEO

你可以：

- 在 CEO 控制台查看会议报告
- 看到员工正在开会的状态（他们会出现在办公室视图的会议室中）
- 会议不需要你的批准 — 员工按需自行组织
