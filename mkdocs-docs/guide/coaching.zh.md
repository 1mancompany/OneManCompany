# 1-on-1 辅导

OneManCompany 最强大的功能之一：CEO 辅导会话，永久性地塑造员工的工作方式。

## 工作原理

当你通过 1-on-1 会话辅导员工时：

1. 你选择一位员工并开始辅导会话
2. 你提供反馈、方向指引或新的工作原则
3. 系统将你的指导提炼到员工的**工作原则**（`work_principles.md`）中
4. 从此以后，该员工的行为会体现你的辅导内容

这不是一次性的临时指令 — 而是永久性的行为改变，就像现实中导师逐步塑造一个初级员工一样。

## 你可以辅导什么

- **工作风格** — "报告要更简洁"、"始终包含测试用例"
- **技术方向** — "优先使用函数式范式而非 OOP"、"严格使用 TypeScript"
- **沟通方式** — "每次会议结束时总结决议"
- **质量标准** — "没有单元测试绝不发布"、"始终考虑边界情况"
- **领域知识** — "我们的用户是非技术人员，保持 UX 简洁"

## 如何持久化

你的指导会写入员工的 `work_principles.md` 文件：

```
employees/
└── 00006/
    ├── profile.yaml
    ├── work_principles.md    ← 你的辅导内容存在这里
    └── guidance.yaml
```

每次员工执行任务时，工作原则都会注入到其推理上下文中。员工在你辅导之后，思考方式会真正发生改变。

## 高效辅导的建议

!!! tip "要具体"
    不要说"写更好的代码"，而要说"所有网络请求都要加错误处理，重试前记录错误详情"。

!!! tip "在评审工作成果后辅导"
    最佳辅导时机是在评审任务交付物之后。结合实际产出的具体反馈，比抽象的原则更有效。

!!! tip "累积效应"
    每次辅导会话都建立在之前的基础上。随着时间推移，你的员工会形成对你的标准和偏好的深刻理解。

## 组织进化

辅导不仅仅是个人提升。跨团队的累积效应推动整个组织进化：

- 员工层面的洞察沉淀为**工作原则**
- 项目层面的经验沉淀为**公司知识库**
- 两者结合，你的 AI 公司在每个项目后都变得更好

这就是为什么 OneManCompany 是一个操作系统，而不仅仅是一个工具 — 组织本身在学习和进步。
