# 项目承接工作流

- **流程ID**: project_intake
- **责任人**: COO（铁面侠）
- **协作人**: HR（知心姐）、相关员工
- **触发条件**: CEO下达新项目任务

---

## 阶段一：任务描述总结

- **负责人**: COO
- **步骤**:
  1. 接收CEO下达的项目任务
  2. 分析任务需求，拆解为可执行的子任务
  3. 撰写项目概述文档，明确项目背景、范围和约束条件
- **产出**: 项目概述文档

## 阶段二：目标制定

- **负责人**: COO
- **步骤**:
  1. 根据项目概述制定可量化的目标（SMART原则）
  2. 确定项目里程碑和交付时间节点
  3. 明确验收标准
- **产出**: 项目目标与里程碑文档

## 阶段三：人员分配

- **负责人**: HR
- **步骤**:
  1. 根据项目需求评估现有人员能力
  2. 分配项目成员，指定项目负责人
  3. 如果人手不足，启动「人员招聘工作流」
  4. 确认每位成员的职责分工
- **产出**: 项目人员分配表

## 阶段四：项目执行与跟踪

- **负责人**: 项目负责人（由COO指定）
- **步骤**:
  1. 各成员按分工执行任务
  2. COO定期检查进度，对照里程碑
  3. 遇到阻碍时向COO申请资源或调整计划
- **产出**: 进度更新记录

## 阶段五：项目验收

- **负责人**: COO
- **步骤**:
  1. 对照验收标准检查交付物
  2. 组织相关人员进行质量评审
  3. 高级别员工抽查低级别员工的工作产出
  4. 不通过则返回修改，通过则进入下一阶段
- **产出**: 验收报告

## 阶段六：项目报告撰写

- **负责人**: COO
- **协作人**: 项目全体成员
- **步骤**:
  1. 汇总项目过程中的关键数据和成果
  2. 各成员提交个人贡献总结
  3. COO撰写完整项目报告
  4. 报告提交CEO审阅
- **产出**: 项目最终报告
- **后续**: 自动触发「项目复盘工作流」
