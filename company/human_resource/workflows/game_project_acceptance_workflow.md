# 游戏项目验收工作流

1. **检查交付物位置**：确认代码文件已成功保存在指定的项目workspace中，并列出关键文件清单。
2. **验证代码可运行性与功能**：
   - 审查主要代码文件（如HTML、JS），确认其运行环境和依赖（如无需额外后端服务即可在浏览器端运行）。
   - **功能验证**：逐一测试游戏核心机制（如发射机制、物体交互、计分系统、关卡判定、胜负判定等）。
3. **逐条对照验收标准**：
   - 将每个验收标准逐一与实际输出对比，记录通过/不通过。
   - 确保所有功能点、代码完整性和文件路径均符合要求。
4. **做出验收决定**：
   - 全部通过：调用 `accept_project(accepted=true)`，附上详细的验证证据。
   - 任一不通过：调用 `accept_project(accepted=false)`，详细列出不通过项和具体问题，要求整改后重新提交。
