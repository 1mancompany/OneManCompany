# 面试评估与题库

## 一、 评估维度与权重

- **基础知识 (30%)**: 语言特性、数据结构、算法、数据库基础。
- **工程化能力 (30%)**: 架构设计、代码规范、测试、CI/CD、性能优化。
- **AI/LLM 经验 (20%)**: 提示词工程、模型调用、LangChain/LlamaIndex 经验（加分项，无经验则考察学习潜力）。
- **协作与软技能 (20%)**: 沟通表达、独立解决问题能力、创业公司文化契合度。

## 二、 面试题库

### 1. 基础知识 (Basic)
- **Python/JS**:
  - Python 中的 GIL 是什么？它对并发编程有什么影响？
  - 解释 Python 的装饰器原理，并手写一个计算函数执行时间的装饰器。
  - JavaScript 中的事件循环 (Event Loop) 是如何工作的？
- **数据结构与算法**:
  - 给定一个数组，找出其中和为目标值的两个数，并返回它们的索引（Two Sum）。要求时间复杂度 O(n)。
  - 如何实现一个 LRU 缓存？
- **数据库**:
  - 解释数据库索引的原理（B+ 树）。
  - 什么是 SQL 注入？如何防范？
  - 描述一次你做过的复杂 SQL 查询优化经历。

### 2. 工程化能力 (Engineering)
- **架构设计**:
  - 如果让你从零开始设计一个短链接生成系统，你会怎么设计？（考察高并发、存储、唯一性）
  - 微服务架构与单体架构的优缺点是什么？在什么阶段应该进行微服务拆分？
- **后端框架**:
  - 描述 FastAPI 的依赖注入系统。它有哪些优势？
- **代码与运维**:
  - 你平时是如何编写单元测试的？如何保证测试覆盖率？
  - 简述你熟悉的一种 CI/CD 流程，以及常用的工具（如 GitHub Actions, GitLab CI）。

### 3. AI/LLM 相关 (AI/LLM Bonus)
- 你有在实际项目中使用过大语言模型（LLM）API 吗？遇到了哪些挑战？（如延迟、Token 限制、幻觉等）
- 解释什么是 RAG（Retrieval-Augmented Generation）？它的核心流程是什么？
- 如果让你设计一个基于 LLM 的代码审查（Code Review）助手，你会如何设计 Prompt 和系统架构？

### 4. 协作与软技能 (Soft Skills)
- **独立作战**: 描述一个你独立负责并成功上线的项目，你在其中遇到了最大的困难是什么？是如何克服的？
- **团队协作**: 当你和团队成员在技术方案上产生严重分歧时，你会如何处理？
- **文化契合**: 为什么想加入一家创业公司？你对未来的职业规划是什么？
