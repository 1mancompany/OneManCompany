# 招聘渠道与候选人搜寻计划（Engineer）

## 1) 招聘渠道组合（按性价比排序）
1. **内推/朋友圈/前同事网络**：速度快、匹配度高；建议 CEO/团队成员同步发“短 JD”。
2. **技术社区**：
   - GitHub（活跃 repo/PR 贡献者）、StackOverflow、HuggingFace Spaces/Transformers 贡献者
   - 相关社区：FastAPI、LangChain/LlamaIndex、RAG/向量检索实践者
3. **内容平台**：
   - 掘金/知乎/公众号（发布 OMC 的工程实践文章，吸引自驱型工程师）
4. **招聘平台**：BOSS 直聘/拉勾/猎聘（按“AI 应用工程/后端工程/Agent 工程师”关键词）
5. **海外**：LinkedIn + Upwork/Toptal（若接受合同制远程）

## 2) 候选人画像关键词（用于检索）
- Python / FastAPI / asyncio / SQLAlchemy / PostgreSQL
- LLM / function calling / tool calling / prompt engineering
- RAG / embeddings / pgvector / Milvus / FAISS
- CI/CD / Docker / GitHub Actions

## 3) 主动搜寻（Outbound）策略
- **目标池**：
  - 近 6–12 个月有 LLM 应用落地文章/项目的人
  - 做过“从 0 到 1 + 上线”的后端工程师
- **触达节奏**：
  - Day 1：首轮私信（简短说明业务+技术栈+影响力）
  - Day 3：补充案例/技术博客链接
  - Day 7：最后一次 follow-up

## 4) 信息包（对外统一话术要点）
- 你会负责：后端 + Agent workflow 的核心交付，上线即产生业务影响。
- 你将获得：充分授权、快速迭代、工程标准可自己主导。
- 我们看重：能把功能做成“可维护、可观测、可迭代”的产品。

## 5) 候选人搜寻（Boss Online MCP）
- 通过人才市场生成候选人包并给 CEO 做卡片筛选（本项目已生成初轮候选）。
