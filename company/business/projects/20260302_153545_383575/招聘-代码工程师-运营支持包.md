# One Man Company｜代码工程师招聘（运营侧）支持包

> 目标：配合 CEO 快速、稳定、可复用地完成代码工程师招聘与入职准备。内容覆盖：**面试组织与日程机制**、**入职前 IT/账号/设备清单与预算**、**外包/猎头供应商与成本区间建议**。

---

## 1) 面试组织与日程机制

### 1.1 面试流程（建议标准版：5–8 个自然日闭环）

| 阶段 | 负责人 | 目标 | 建议时长 | 产出 |
|---|---|---:|---:|---|
| 简历筛选 | CEO/技术面试官A（或指定工程师） | 判断匹配度，避免无效面试 | 10–15 min/份 | 筛选结论（过/不过/待观察） |
| 15min 预沟通（可选） | CEO/COO | 核对动机、薪资区间、到岗时间 | 15 min | 关键约束记录 |
| 技术一面（基础能力） | 技术面试官A | 语言基础、工程习惯、调试能力 | 45–60 min | 评分卡+结论 |
| 技术二面（系统/项目） | CEO/技术面试官B | 系统设计、项目深挖、质量意识 | 60 min | 评分卡+结论 |
| 文化/协作面（小团队适配） | CEO/COO | 沟通、ownership、抗压、价值观 | 30–45 min | 评分卡+结论 |
| 终面/Offer沟通 | CEO | 薪资、岗位期望、入职时间 | 30 min | 口头意向+下一步 |

> 若招聘的是**初级工程师**：减少到「技术一面 + 项目/代码走查 + 文化」即可。


### 1.2 角色分工（RACI）

| 事项 | CEO | COO | 技术面试官/工程师 | HR（如有） |
|---|---|---|---|---|
| 面试官池与面试题库确认 | A | C | R | C |
| 候选人推进节奏与排期 | C | A/R | C | R |
| 会议室/线上会议创建 | C | A/R | C | R |
| 模板/评分卡/记录规范 | C | A/R | C | C |
| Offer审批与发放 | A/R | C | C | R |
| 入职前账号/设备准备 | C | A/R | C | R |

A=Accountable，R=Responsible，C=Consulted。


### 1.3 排期机制（强制 SLA：48h 内推进到下一步）

**原则**：候选人体验=竞争力；小公司必须“快”。

- **SLA-1：简历进入后 24h 内给出初筛结论**（过/不过/待观察）。
- **SLA-2：通过初筛后 48h 内完成至少一轮技术面**。
- **SLA-3：每轮面试结束后 12h 内提交评分卡**；24h 内给候选人反馈是否进入下一轮。
- **SLA-4：终面后 24h 内给出 offer/拒绝结论**。

**排期规则（减少扯皮）**：
1. 预先固化面试官时间块（每周固定 2–3 个时间段）。例如：
   - 周二/周四 20:00–22:00（线上）
   - 周六 10:00–12:00（线上/线下）
2. 同一候选人尽量在 **同一天串联 2 轮**（降低流失）。
3. “候选人优先”策略：在可行范围内，优先匹配候选人时间。


### 1.4 会议室/线上工具配置建议

**线下面试**：按公司会议室预约制度执行（由 COO 统一创建与管理预约）。

**线上面试（推荐默认）**：
- 国内优先：**腾讯会议 / 飞书会议 / 钉钉会议**
- 国际候选人：**Google Meet / Zoom**

**最低配置**：
- 固定会议链接模板（含候选人姓名+岗位+轮次）
- 录制策略（可选）：需提前告知候选人并征得同意；默认不录制
- 共享协作文档：Notion/飞书文档/Google Docs（二选一，统一）


### 1.5 面试资料与模板（运营侧统一发放）

**必须统一的 4 类模板**：
1. 面试邀请（邮件/IM）模板
2. 面试评分卡（按能力维度打分）
3. 面试官复盘/决策记录（debrief）
4. Offer/入职信息收集表

> 具体模板见本文「附录 A」。


### 1.6 面试数据与看板（建议轻量 ATS）

不必一开始上复杂 ATS。建议用以下任一组合：
- **Notion**：数据库=候选人池；视图=阶段看板；字段=轮次/评分/薪资/入职时间
- **飞书多维表格**：同上（国内更顺手）
- **Trello**：仅看板流转，评分卡另存文档

**看板阶段（列）建议**：
- New（待筛）→ Screened（通过初筛）→ Tech1 → Tech2 → Culture → Offer → Hired / Rejected / On hold

**核心指标（每周看一次即可）**：
- 漏斗转化率：简历→初筛→一面→二面→offer
- 平均推进时长（Lead time）
- 拒绝原因 Top3（薪资、技术不匹配、到岗时间…）

## 2) 入职前 IT / 账号 / 设备需求清单与预算建议

### 2.1 入职前时间线（建议：T-7 到 T+7）

| 时间点 | 事项 | 负责人 |
|---|---|---|
| T-7（发 offer 后） | 收集信息：姓名证件、邮箱、常用系统、电脑偏好（Mac/Win/Linux） | COO（或HR） |
| T-5 | 采购/调拨设备；创建基础账号 | COO |
| T-2 | 权限最小化配置（Git/Repo/文档/项目） | COO+CEO |
| T-0（入职日） | 设备交付；安全与开发环境初始化 | COO |
| T+1~T+7 | 访问权限补齐；必要工具订阅开通；试用期目标对齐 | CEO+COO |


### 2.2 账号与权限清单（最小可用集）

> 原则：**最小权限** + **可追溯**（谁批准、何时开通）。

**基础账号（Day0 必须）**
- 公司邮箱（如有域名）或统一工作邮箱方案
- 协作：飞书/钉钉/Slack（择一）
- 文档：Notion/飞书文档/Google Workspace（择一）
- 代码：GitHub / GitLab 组织账号
- 密码管理：1Password / Bitwarden（推荐至少上一个）

**研发相关（按需）**
- CI/CD：GitHub Actions / GitLab CI（无需单独账号）
- 云服务：阿里云/腾讯云/AWS（强烈建议用 IAM 子账号，不给 root）
- 监控/日志：Sentry / Grafana Cloud / Datadog（按预算）

**权限审批建议**
- 代码仓库写权限、生产环境权限：必须 CEO 批准
- 仅开发环境权限：COO 可执行，CEO 备案


### 2.3 设备清单（工程师标准配置）

#### 方案 A：Mac 方案（适合全栈/移动/需要类 Unix 环境）
- 笔记本：MacBook Pro 14"（M3/M3 Pro，16GB/512GB 起）
- 外设：
  - 27" 显示器（可选但强烈建议）
  - Type-C 扩展坞
  - 机械键盘/鼠标（可选）

#### 方案 B：Windows/Linux 方案（适合后端/成本敏感）
- 笔记本：ThinkPad / Dell（i7/32GB/1TB SSD 建议）
- 统一安装：WSL2 或 Ubuntu 双系统（视团队栈而定）

#### 通用耗材
- 耳机（带麦，线上面试/线上协作必要）
- 安全：硬盘加密（FileVault/BitLocker）、屏幕锁、2FA


### 2.4 预算建议（人民币，含税大致区间）

| 项目 | 低配（可用） | 标配（推荐） | 说明 |
|---|---:|---:|---|
| 笔记本电脑 | 6,000–9,000 | 12,000–18,000 | Mac 通常在标配区间；Win/Linux 可更低 |
| 显示器 | 800–1,500 | 1,800–3,000 | 生产力显著提升 |
| 扩展坞/线材 | 200–500 | 600–1,200 | Mac 用户建议标配 |
| 耳机 | 150–300 | 400–900 | 线上会议必需 |
| 软件/订阅（每人/月） | 0–100 | 100–300 | 视是否上 1Password、Notion/Workspace 等 |

**建议结论**：
- 以“标配”测算：**每名工程师一次性硬件预算约 15,000–25,000**；持续订阅 **100–300/月**。


### 2.5 入职资料与环境初始化清单（Day0 Checklist）

- [ ] 设备到位并贴资产编号（如有）
- [ ] OS 更新、磁盘加密、屏幕锁、2FA
- [ ] 开发环境：语言运行时、IDE、Docker（如用）、SSH Key
- [ ] Git 组织加入、仓库权限、分支保护规则说明
- [ ] 文档与流程入口：README/Runbook/Onboarding 文档链接
- [ ] 试用期目标与交付物（由 CEO 提供，COO 负责记录与提醒）

## 3) 外包 / 猎头：供应商与成本区间建议

### 3.1 什么时候考虑外包/猎头

**猎头适用**：
- 要找中高级、稀缺栈、需要“人选质量”而不是“简历数量”
- CEO 时间有限，需要对接与初筛外包

**外包适用**：
- 需求明确、可拆分、可验收（例如：某个模块、某个 PoC）
- 短期交付压力大，但不确定是否长期需要该岗位


### 3.2 猎头/招聘渠道建议（中国市场为主）

**渠道（自投放/直招）**
- BOSS 直聘（快、量大；需强筛）
- 拉勾（互联网岗位）
- 猎聘（中高端更强）
- 脉脉（定向挖人、内推）
- LinkedIn（偏国际/外企背景）

**猎头公司（示例类型，实际可由 COO 进一步比价筛选）**
- 国际综合：Robert Walters / Michael Page / Hays（通常更贵，适合中高端）
- 国内综合：猎聘精选猎头/当地精品猎头（性价比更高，交付差异较大）

> 注：供应商最终选型建议以“试单 2–4 周 + 简历质量/响应速度/offer 接受率”评估。


### 3.3 成本区间（行业常见）

#### 3.3.1 猎头费用
- **按成功付费（Contingency）**：通常 **年薪的 15%–25%**
  - 例：年薪 40 万，费用约 6–10 万
- **保留制（Retained）**：常见 30% 左右，分期付款；适合高端岗位

**合同关键条款建议**
- 保证期：90 天（常见），期间离职免费重推
- 候选人排他：避免“多家猎头同一候选人撞单”争议
- 简历归属期：6–12 个月常见，需确认

#### 3.3.2 外包/外部开发（人天/人月）
- 国内外包开发（个人/工作室）：**800–2,000 元/人天** 常见（视栈与资历）
- 中型外包公司：**1,200–3,000 元/人天**（含项目管理溢价）

**外包风险控制（必须）**
- 明确验收：里程碑、代码质量（Lint/测试/审查）、交付物清单
- 知识产权与保密：合同必须写清
- 代码托管在公司仓库，外包只拿最小权限


### 3.4 建议的选择策略（可执行）

1. **先直招 2 周**：BOSS/拉勾/内推，积累候选人画像与面试题校准。
2. 若 2 周后仍无高质量候选人进入二面：
   - 选 1–2 家猎头“试单”（明确画像、薪资、工作方式、到岗时间）。
3. 如业务有明确短期交付缺口：并行找外包做可验收模块，CEO 保留核心架构与关键路径。
