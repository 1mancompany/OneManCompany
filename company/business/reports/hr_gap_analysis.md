# OneManCompany HR 功能差距分析报告

> 基于行业标准人力资源管理六大模块，对照本系统 HR Agent 现有能力，梳理缺失功能与改进建议。

---

## 一、现有功能总览

| 模块 | 已实现功能 | 实现程度 |
|------|-----------|---------|
| 招聘与配置 | 岗位需求列表、候选人搜索/筛选、CEO面试、入职创建 | ★★★☆☆ |
| 绩效管理 | 季度评分(3档)、绩效历史、自动晋升 | ★★☆☆☆ |
| 员工关系 | 解雇流程、前员工管理、重新雇佣 | ★★☆☆☆ |
| 组织管理 | 部门分区、层级体系、花名文化 | ★★★☆☆ |
| 培训开发 | ❌ 基本未实现 | ☆☆☆☆☆ |
| 薪酬福利 | ❌ 完全缺失 | ☆☆☆☆☆ |

## 二、按模块详细分析缺失功能

### 模块 1：人力资源规划（完全缺失）

当前系统没有任何前瞻性的人力规划能力，HR 只能被动响应 CEO 指令。

| 缺失功能 | 说明 | 优先级 |
|----------|------|--------|
| **人力编制规划** | 根据公司业务目标，规划各部门所需人数上限和岗位构成 | 🔴 高 |
| **人才缺口分析** | 自动对比现有员工技能与业务需求，识别能力缺口 | 🔴 高 |
| **人力成本预算** | 跟踪每位员工的 LLM 调用成本（token 消耗），编制人力预算 | 🟡 中 |
| **组织架构图** | 可视化呈现公司组织架构、汇报关系、部门编制 | 🟡 中 |
| **岗位说明书** | 每个岗位的职责描述、任职资格、能力模型（目前只有粗略的 role 字段） | 🟡 中 |
| **人才梯队建设** | 为关键岗位识别后备人选，制定继任计划 | 🟢 低 |

### 模块 2：招聘与配置（部分实现，有明显缺口）

已有基础招聘流程，但缺少多个关键环节。

| 缺失功能 | 说明 | 优先级 |
|----------|------|--------|
| **结构化面试** | 当前面试是自由问答，缺少标准化评分表和面试题库 | 🔴 高 |
| **多轮面试流程** | 只有 CEO 单轮面试，缺少技术面（同部门 Senior）+ HR 面 + CEO 终面的多轮机制 | 🔴 高 |
| **试用期管理** | 新员工没有试用期概念，入职即正式，缺少试用期评估和转正流程 | 🔴 高 |
| **内部调岗/转岗** | 不支持员工在部门间调动或角色转换 | 🟡 中 |
| **招聘数据分析** | 缺少招聘漏斗统计（投递数→筛选数→面试数→录用数→到岗数） | 🟡 中 |
| **人才库管理** | 未录用的优秀候选人没有存入人才库，无法后续召回 | 🟡 中 |
| **招聘渠道评估** | Boss Online 是唯一渠道，缺少渠道效果评估和多元化招聘来源 | 🟢 低 |
| **背景调查** | 缺少候选人背调环节（在 AI 公司语境下可模拟为"能力验证测试"） | 🟢 低 |

### 模块 3：培训与开发（基本缺失）

系统中 `skills/*.md` 文件只是静态标签，没有真正的培训和成长体系。

| 缺失功能 | 说明 | 优先级 |
|----------|------|--------|
| **入职培训流程** | workflow 中定义了培训阶段但未真正执行——新员工没有熟悉公司/工具的过程 | 🔴 高 |
| **技能成长系统** | 员工的 skills 是静态的，不会随工作经验增长或变化 | 🔴 高 |
| **导师制（Mentor）** | 招聘 workflow 提到"分配 mentor"但系统未实现 mentor-mentee 配对 | 🔴 高 |
| **培训计划制定** | HR 不能为员工制定个性化学习/提升计划 | 🟡 中 |
| **培训效果评估** | 缺少培训后的能力测试或效果验证 | 🟡 中 |
| **知识库系统** | 员工积累的项目经验没有沉淀为可复用的组织知识 | 🟡 中 |
| **技能认证体系** | 员工学会新技能后没有认证/更新 profile 的机制 | 🟡 中 |
| **跨部门轮岗** | 不支持员工到其他部门短期轮岗学习 | 🟢 低 |

### 模块 4：绩效管理（框架已有，深度不足）

现有 3 档评分 + 自动晋升是一个起点，但远不够完善。

| 缺失功能 | 说明 | 优先级 |
|----------|------|--------|
| **目标管理（OKR/KPI）** | 员工没有明确的工作目标，绩效评分缺乏客观依据 | 🔴 高 |
| **多维度评价** | 当前只有 HR 单方面打分，缺少 360 度评估（自评 + 同事互评 + 上级评 + 下级评） | 🔴 高 |
| **绩效面谈** | 评分后没有一对一绩效沟通，员工不知道自己为何得这个分 | 🔴 高 |
| **绩效改进计划（PIP）** | 低绩效员工（3.25）没有改进计划，直接等着被淘汰 | 🔴 高 |
| **评分维度细化** | 3 档评分太粗（3.25/3.5/3.75），无法区分不同维度（工作质量、协作、创新等） | 🟡 中 |
| **实时绩效反馈** | 只有季度评审，缺少日常的即时反馈/表扬机制 | 🟡 中 |
| **任务完成质量评估** | current_quarter_tasks 只计数量，不评价质量 | 🟡 中 |
| **绩效与任务关联** | 评分时 HR 看不到员工具体做了什么任务，只知道完成了 3 个 | 🟡 中 |
| **降级机制** | 连续低绩效没有降级处理，只有解雇 | 🟢 低 |

### 模块 5：薪酬与福利（完全缺失）

当前系统没有任何薪酬概念，AI 员工"无偿工作"。

| 缺失功能 | 说明 | 优先级 |
|----------|------|--------|
| **薪资结构** | 缺少薪资等级表（salary band），不同 level/role 没有对应的薪资范围 | 🟡 中 |
| **LLM 成本作为"薪资"** | 每个员工的 LLM 调用是真实成本，可映射为"工资"进行跟踪和预算 | 🟡 中 |
| **薪资调整机制** | 晋升时没有加薪，绩效优秀时没有奖金 | 🟡 中 |
| **福利体系** | 没有任何福利概念（工具使用权限可以视为一种"福利"但未体系化） | 🟢 低 |
| **薪酬公平性分析** | 不同部门/角色之间是否存在不合理的成本差异 | 🟢 低 |

> **注**：在 AI 公司的语境下，"薪酬"可以创造性地映射为 **LLM token 消耗预算**——每个员工使用的模型价格 × 调用量 = 人力成本。这是一个独特且有趣的机制。

### 模块 6：员工关系（框架已有，缺少深度）

有解雇和重雇佣，但员工关系管理远不止这些。

| 缺失功能 | 说明 | 优先级 |
|----------|------|--------|
| **入职 Onboarding 清单** | 新员工入职后没有系统性的引导流程（介绍同事、熟悉工具、了解规章） | 🔴 高 |
| **离职 Offboarding 流程** | 解雇时没有：交接工作、回收权限、离职面谈、知识转移 | 🔴 高 |
| **离职面谈** | 员工离职前不收集反馈，失去改进公司管理的重要信息来源 | 🔴 高 |
| **员工满意度调查** | 没有定期调查员工对公司/管理/工作环境的满意度 | 🟡 中 |
| **员工申诉机制** | 员工无法主动反映问题或对绩效评定提出异议 | 🟡 中 |
| **工时与考勤** | 没有跟踪员工工作时长、响应速度等出勤数据 | 🟡 中 |
| **请假管理** | 没有请假/休假概念（可映射为"agent 暂停服务/降低频率"） | 🟢 低 |
| **员工关怀** | 没有生日祝福、入职周年纪念、团建活动等关怀机制 | 🟢 低 |
| **劳动纠纷处理** | 当多个员工在会议中产生冲突时，缺少调解机制 | 🟢 低 |

## 三、系统性缺失（跨模块问题）

除六大模块的具体功能外，以下系统性能力也是缺失的：

### 3.1 HR 数据分析与报表

| 缺失项 | 说明 |
|--------|------|
| HR 仪表盘 | 缺少 HR 专属的数据看板（人员流动率、部门分布、绩效趋势等） |
| 人力分析报告 | HR 无法自动生成人力资源分析报告 |
| 预测性分析 | 不能预测离职风险、招聘需求趋势 |
| 人效分析 | 缺少每个员工/部门的投入产出比分析 |

### 3.2 HR 自治能力

| 缺失项 | 说明 |
|--------|------|
| HR 主动决策 | HR 完全被动——只响应 CEO 指令或关键词路由，不能主动发现问题并提出建议 |
| 定期巡检 | HR 不会主动检查员工状态、发现潜在问题（如长期低绩效、技能瓶颈） |
| 制度建议 | HR 不能基于数据向 CEO 建议调整人力政策 |

### 3.3 员工生命周期完整性

当前员工生命周期存在断裂：

```
现状:  招聘 → 入职(简) → 工作 → 评审 → [晋升/解雇]
                                          ↑ 缺口很大

应有:  人力规划 → 招聘 → 面试(多轮) → 入职培训 → 试用期 → 转正
       → 目标设定 → 日常工作 → 持续反馈 → 季度评审 → 绩效面谈
       → [晋升/调岗/培训/PIP] → ... → [主动离职/辞退] → 离职面谈
       → 知识转移 → 权限回收 → 人才库归档
```

## 四、优先级排序：推荐实施路线图

### 第一阶段：基础补全（高优先级）

1. **目标管理（OKR）** — 让绩效评分有依据
2. **入职培训流程** — 新员工需要系统化引导
3. **试用期管理** — 降低招聘风险
4. **离职面谈 + Offboarding** — 完善员工退出流程
5. **绩效面谈** — 评分后需要与员工沟通
6. **绩效改进计划（PIP）** — 给低绩效员工改进机会
7. **多轮面试** — 提高招聘质量
8. **技能成长系统** — 员工能力应随经验提升

### 第二阶段：能力增强（中优先级）

9. **人力编制规划** — HR 主动规划人力需求
10. **人才缺口分析** — 自动识别技能缺口
11. **360度评估** — 多角度评价更客观
12. **员工满意度调查** — 掌握组织健康度
13. **内部调岗** — 人才流动与优化配置
14. **导师制** — 加速新人成长
15. **LLM成本跟踪（薪资映射）** — 量化人力成本
16. **知识库沉淀** — 组织知识资产管理

### 第三阶段：高级特性（低优先级）

17. 人才库管理（未录用候选人召回）
18. 员工关怀机制
19. 薪酬公平性分析
20. 预测性 HR 分析
21. 跨部门轮岗
22. 降级机制

## 五、特别建议：AI 公司的独特 HR 创新点

OneManCompany 是一个 AI 员工组成的公司，有些 HR 功能可以做出超越传统 HR 系统的创新：

| 创新方向 | 说明 |
|----------|------|
| **LLM 模型即"学历/能力"** | 员工使用的 LLM 模型直接决定能力上限，"培训"可以映射为升级模型 |
| **Prompt Engineering 即"培训"** | 优化员工的 system_prompt 和 work_principles 就是在"培训"员工 |
| **Token 消耗即"薪资"** | 真实的 API 成本可作为薪资系统的基础，CEO 需要控制"人力预算" |
| **技能文件即"能力证书"** | skills/*.md 文件可以动态更新和验证，作为能力认证体系 |
| **Agent 温度即"工作风格"** | temperature 参数可以视为员工的"工作风格"（保守 vs 激进） |
| **会议记录即"工作日志"** | 所有会议自动记录，可作为绩效评估的客观依据 |
