# 一页式任务模板 (One-Page Task Template)

## 1. 任务背景与目标 (Context & Objective)
- **项目名称**: 愤怒的小鸟游戏开发
- **宏观指令**: 生成一个愤怒的小鸟游戏，鸟有鸟的图标，木头要有耐久，要有关卡和计分。
- **业务目标**: 完成核心玩法MVP版本的开发，包含基础物理引擎、UI交互、多关卡支持及计分系统。

## 2. 任务拆解与路由分派 (Task Breakdown & Routing)
| 子任务 | 负责高管/员工 | 产出要求 | 验收标准 |
|---|---|---|---|
| 核心物理与逻辑开发 | Engineer (00010) | 游戏主逻辑、耐久度机制、计分系统 | 代码无报错，逻辑符合物理预期 |
| 视觉与UI设计 | Designer (00009) | 鸟的图标、木头/障碍物素材、UI界面 | 素材尺寸适配，风格统一 |
| 项目统筹与集成 | COO (00003) | 整合代码与素材，构建关卡，最终交付 | 游戏可顺畅运行，包含至少2个关卡 |

## 3. 资产与版本控制规范 (Asset & Version Control)
- **Workspace**: 统一在当前项目 `workspace` 目录下进行产出。
- **版本号规范**: v0.1.0 (基础框架) -> v0.5.0 (素材集成) -> v1.0.0 (MVP交付)。
- **变更记录**: 所有代码与素材的变更必须在 `changelog.md` 中记录。

## 4. 预算与资源 (Budget & Resources)
- **预估预算**: $0.15
- **依赖工具**: Version Control and Asset Manager, OpenSandbox
